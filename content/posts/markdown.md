# ブロック要素とスパン要素について

markdownの記法は大きくブロック要素とスパン要素に分けられます。ブロック要素はその段落中に有効で、空行を挟むことで有効範囲を区切ります。スパン要素は特定の部分に使用し、記号等で囲む形で使用します。

段落自体もブロック要素であり、プレーンなブロック要素が段落であると解釈できます。

# ブロック要素の記法

## 段落

段落は1行空けて表現します。空けずに次の行を記述した場合は1つの段落として扱われ、改行されません。

``` text
第一段落
この行も第一段落として扱われます

第二段落
```

第一段落
この行も第一段落として扱われます

第二段落

----------------

段落内で明示的に改行する場合には行末に2つのスペースを挟んで改行します。

``` text
第一段落
前行でスペースを挟んでないので改行されません  
前行でスペースを挟んだので改行されます

第二段落
```

第一段落
前行でスペースを挟んでないので改行されません  
前行でスペースを挟んだので改行されます

第二段落

## 引用

引用にはブロック引用符>とスペースを使用します。引用ブロックはネスト可能です。ちょうど電子メールと同じ様に表現出来ます。

``` text
> 引用ブロックです
> ## ヘッダも機能します
> > ネストされた引用ブロックです
>
> ネストから復帰します
```

> 引用ブロックです
> #### ヘッダも機能します 
> > ネストされた引用ブロックです
> 
> ネストから復帰します

----------------

ブロック要素の仕様上、引用ブロック(もしくはネスト)の後には空白行(もしくはネストレベルを下げた空の行)を挟まないと判定を誤ります。

``` text
> 引用ブロックです
この行も引用ブロック判定されます

空白行を挟むと引用ブロック判定されません

> 引用ブロックです
> > ネストされた引用ブロックです
> ネスト後に空の行を挟まないとこの行もネスト判定されます
```

> 引用ブロックです
この行も引用ブロック判定されます

空白行を挟むと引用ブロック判定されません

> 引用ブロックです
> > ネストされた引用ブロックです
> ネスト後に空の行を挟まないとこの行もネスト判定されます

## リスト(箇条書き)

番号なしリスト、番号つきリスト、定義リストがサポートされています。

### 番号なしリスト

番号なしリストにはハイフン、アスタリスク、プラスの次にスペースを使用します。**自分はハイフンが好き**。リストはブロック要素で、アイテムの判別はインデントレベルで行われます。

``` text
- 箇条書き
+ これも箇条書き
* これもこれも箇条書き
```

- 箇条書き
+ これも箇条書き
* これもこれも箇条書き

----------------

4つのスペースでインデントすることで箇条書きをネストすることが出来ます。

``` text
- 箇条書き
    - ネストされた箇条書き
- これも箇条書き
```

- 箇条書き
    - ネストされた箇条書き
- これも箇条書き

### 番号つきリスト

番号つきリストには番号、ピリオドの次にスペースを仕様します。ただし番号はhtmlタグの仕様上機能しません。

``` text
1. hoge
2. fuga
    1. hogehoge
    2. fugafuga
    3. piyopiyo
3. piyo
```

1. hoge
2. fuga
    1. hogehoge
    2. fugafuga
    3. piyopiyo
3. piyo

### 定義リスト

定義リストは1行の用語とそれに続く定義からなります。定義はコロンと1つ以上のスペースから始めます。次の用語と定義の間は空白行で区切る必要があります。

``` text
hoge
: hogeとは何か

fuga
: fugaとは何か
: fugaの初出

piyo
piyopiyo
: piyoとは何か
```

# スパン要素

## リンクと画像

単純なリンクは次の様に作成できます。

``` text
<https://blog.rkgknet.org>
```

<https://blog.rkgknet.org>

リンクのテキストを変更するにはテキストを角括弧で囲み、URLを括弧で囲みます。

``` text
ようこそ、[rkgknet](https://blog.rkgknet.org)へ!
```

ようこそ、[rkgknet](https://blog.rkgknet.org)へ!

URLが長くて文章構成に邪魔な場合、別の場所で参照リンクを定義することもできます。 この時角括弧を1つにすると、リンクテキストと参照名が等価になります。ただし、参照の定義に関してはブロック要素になります。

``` text
ようこそ[我がブログ][blog]へ!

ようこそ[blog]へ!

[blog]: https://rkgknet.org "トップページ"
```

ようこそ[我がブログ][blog]へ!

ようこそ[blog]へ!

[blog]: https://rkgknet.org "トップページ"

## 強調

1つのアスタリスクで囲まれた部位は強調されます。囲む文字を2つにすることで更に強調されます。強調はイタリックと同義であり更に日本語フォントのイタリックは斜体であることがほとんどです。斜体は強調の意をあまり成さないので**日本語を使用する限り基本強調はアスタリスク2つを使う**。

``` text
どうだ、*強調*されているだろう。

おや、**更に強調**されているんじゃないか?
```

どうだ、*強調*されているだろう。

おや、**更に強調**されているんじゃないか?

## インラインコード

バッククオートで区切ることで文章中でコードを表現することができます。また、コード内でバッククオートを使用する場合、区切り文字を2つ以上にすることで内部のバッククオートがエスケープされます。要素開始直後と終了直前のスペースは無視されます。

``` text
次に`echo "bionic" > .config`を実行してください

次に`` echo `lsb_release -cs` > .config ``を実行してください
```

## 脚注

脚注の記法は参照リンクの定義と似ています。角括弧、キャレット、脚注名で構成される脚注マーカーをスパン要素として配置し、テキストのどこかにブロック要素として脚注の定義を記述します。脚注の定義に複数のブロックを含む場合には4つのスペースかタブでインデントする必要があります。

~~~ text
脚注[^1]はこの様に書きます

[^1]: 
    脚注はここに定義します

    > ブロック引用も
    > ちゃんと脚注に含まれます
~~~
